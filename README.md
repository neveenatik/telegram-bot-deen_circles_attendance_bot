# بوت الحضور – Telegram Attendance Bot

بوت تيليغرام بالعربية لتسجيل الحضور وتتبع الغيابات.

يدعم النشر على Vercel مع Supabase في الإنتاج.

## المتطلبات

- Node.js 22+
- حساب بوت على Telegram عبر [@BotFather](https://t.me/BotFather)
- حساب Supabase إذا كنت ستنشر على Vercel
- يجب إضافة البوت إلى مجموعة Telegram. الأوامر الإدارية تعمل فقط من أعضاء المجموعة الذين هم مشرفون في Telegram.
- **لا يحتاج البوت نفسه إلى صلاحية مشرف** للعمل. وإذا مُنح صلاحية مشرف فسيثبّت قائمة الحضور تلقائياً (ميزة اختيارية)؛ وإن لم يُمنح تعمل كل الوظائف الأخرى دون تثبيت.

## الإعداد

```bash
# 1. أنشئ نسخة محلية من المشروع وانتقل إلى مجلده
cd telegram-bot

# 2. فعّل نسخة Node المطابقة للمشروع
nvm use

# 3. ثبّت الحزم
npm install

# 4. أنشئ ملف .env من القالب
cp .env.example .env

# 5. عدّل .env وأضف القيم المطلوبة
#    BOT_TOKEN=                 ← من @BotFather
#    SUPABASE_URL=              ← Project URL من Supabase
#    SUPABASE_SERVICE_ROLE_KEY= ← service_role secret من Supabase
#    WEBHOOK_SECRET=            ← قيمة عشوائية اختيارية للأمان
#    FEEDBACK_GROUP_ID=         ← معرّف المجموعة لاستقبال الرسائل (رقم سالب للمجموعات)

# 6. أضف البوت إلى المجموعة (لا يحتاج إلى صلاحية مشرف؛ ومنحها اختياري لتثبيت القائمة)
#    الأوامر الإدارية تعمل فقط داخل المجموعة ومن المشرفين في Telegram

# 7. شغّل البوت
npm start

# 8. (اختياري) سجّل قائمة الأوامر لظهور autocomplete في Telegram
npm run set-commands
```

## النشر على Vercel + Supabase

إذا أردت تشغيل البوت على Vercel بدل التشغيل المحلي، اتبع الخطوات التالية:

1. أنشئ مشروعاً جديداً في Supabase.
2. نفّذ ملف [supabase_v2.sql](scripts/supabase_v2.sql) داخل SQL Editor لإنشاء جداول مخطط V2.
3. انسخ `SUPABASE_URL` من الصفحة الرئيسية للمشروع في Supabase.
4. انسخ `SUPABASE_SERVICE_ROLE_KEY` من Settings → API → Legacy anon, service_role API keys.
5. ارفع المشروع إلى GitHub ثم اربطه مع Vercel.
6. أضف متغيرات البيئة في Vercel:
    - `BOT_TOKEN`
    - `SUPABASE_URL`
    - `SUPABASE_SERVICE_ROLE_KEY`
    - `WEBHOOK_SECRET` إذا كنت تستخدم التحقق من webhook
    - `FEEDBACK_GROUP_ID` إذا كنت تريد استقبال رسائل التغذية الراجعة
7. بعد النشر، سجّل قائمة أوامر Telegram حتى يظهر autocomplete عند كتابة `/`:

```bash
npm run set-commands
```

**ملاحظة:** شغّل هذا الأمر من جهازك المحلي مع متغيرات البيئة الصحيحة. يمكنك أيضاً إعادة تشغيله كلما أضفت أو عدّلت الأوامر في الكود.

8. بعد ذلك اضبط webhook عبر:

```bash
npm run set-webhook -- https://your-project.vercel.app/api/telegram
```

أو استبدل الرابط برابط النشر الفعلي الخاص بك.

**ملاحظة:** شغّل هذا الأمر مرة واحدة فقط بعد نشر المشروع على Vercel. عند تغيير رابط الدومين، أعد تشغيله بالرابط الجديد.

ملاحظة: عند وجود `SUPABASE_URL` و `SUPABASE_SERVICE_ROLE_KEY` سيستخدم البوت Supabase تلقائياً، وإلا سيعود إلى التخزين المحلي داخل مجلد `data/`.

### ملاحظة حول السكريبتات

| السكريبت | متى تستخدمه | الأمر |

### مخطط التخزين الجديد (V2)

- المخطط العلاقي الجديد موجود في [scripts/supabase_v2.sql](scripts/supabase_v2.sql).
- خطة الترحيل المرحلية موجودة في [docs/storage-v2-migration-plan.md](docs/storage-v2-migration-plan.md).

#### ترقية النشرات القائمة

ملف [scripts/supabase_v2.sql](scripts/supabase_v2.sql) محدَّث دائماً بأحدث مخطط، لذا من يُنشئ قاعدة بيانات جديدة **لا يحتاج إلى تشغيل أي migrations**. أمّا النشرات القائمة من قبل تحديثٍ ما فتُرقَّى بتشغيل ملفات الترحيل الموجودة في مجلد [scripts/migrations/](scripts/migrations/) بترتيبها الزمني (اتّبعي سياسة الترحيل الآمن أدناه).

#### سياسة الترحيل الآمن في CI/CD

- اعتبر [scripts/supabase_v2.sql](scripts/supabase_v2.sql) مخطط bootstrap للبيئات الجديدة فقط.
- لا تعتمد على تعديل bootstrap وحده للترقيات على قواعد بيانات قائمة.
- لكل تغيير جديد: أضف ملفاً جديداً داخل `scripts/migrations/` بترتيب زمني.
- اجعل كل migration قابلاً لإعادة التشغيل (idempotent) باستخدام `if not exists` أو جدول تتبع نسخ.
- في pipeline: شغّل migrations بالترتيب قبل نشر التطبيق.
- بعد نجاح migrations فقط، ابدأ deployment لتجنب عدم توافق الكود مع قاعدة البيانات.

#### تدقيق وإصلاح البيانات بعد الترحيل

```bash
# فحص المشاكل فقط (بدون تعديل)
npm run audit-v2:check

# إصلاح تلقائي للمشاكل الشائعة
npm run audit-v2:fix

# تخصيص حد stale-lock (مثال: 5 دقائق)
node scripts/audit-v2-fixes.js --yes --stale-ms=300000
```

يشمل الفحص/الإصلاح:
- إنشاء `group_settings` المفقودة إن وجدت.
- ربط `session_participants` غير المربوطة بعضو عند تطابق الاسم داخل نفس المجموعة.
- تنظيف حقول الحلّ في الطلبات المعلقة `pending` إذا كانت ممتلئة بالخطأ.
- تحويل `processed_updates` العالقة بحالة `processing` إلى `failed` بعد انتهاء مهلة stale-lock.

### تنظيف البيانات غير المستخدمة

بدلاً من حذف بيانات المجموعة فور إزالة البوت منها، يتتبع البوت آخر نشاط لكل مجموعة. يمكنك حذف بيانات المجموعات غير النشطة منذ 90 يوماً أو أكثر عبر:

```bash
# فحص فقط بدون حذف
npm run cleanup-stale-groups:check

# حذف فعلي للمجموعات غير النشطة منذ 90 يوماً
npm run cleanup-stale-groups

# تخصيص عدد الأيام يدوياً
node scripts/cleanup-stale-groups.js --days=30 --yes
```

يشمل الحذف: قوائم الطالبات، الجلسات الحالية والمؤرشفة، التقدّم، المعلمات، وحالة الانتظار الخاصة بالمجموعة.

## سير العمل المعتاد

1. المشرف يرسل `/register` داخل المجموعة.
2. الطالبة تضغط زر "طلب التسجيل" في الرسالة.
3. يحفظ البوت الطلب تلقائياً ضمن الطلبات المعلّقة للمجموعة.
4. المشرف يراجع الطلبات من `/pendingstudents` ثم يضيف الطالبة مباشرة بزر واحد، أو يضيفها يدوياً عبر `/addstudent 123456789 | أحمد محمد`.
5. المشرف يبدأ قائمة: `/startlist اجتماع يونيو`
6. الأعضاء يسجّلون حضورهم من واجهة الأزرار: ✅ حاضرة / 👂 مستمعة / 🔔 معتذرة.
7. أزرار المشرف تبقى ظاهرة طوال الجلسة: «🔄 تحديث»، «✏️ تعديل الحضور» (أو `/editlist`) لمراجعة الحالات وتعديلها، و«⏸️ إيقاف القائمة» لتجميد التسجيل (أو `/freezelist`).
8. المشرف ينهي الجلسة من زر «🛑 إنهاء الجلسة» (أو `/stoplist`) ← يتم تعطيل الواجهة وتوليد التقرير النهائي مع كفارة المجلس.

> **قوائم تصحيح التلاوة** (`/startsecondarylist`): تُسجِّل الطالبة نفسها عبر زرّي الإقرار «✅ حضرتُ الحلقة الأساسية وأُسجّل» أو «📝 لم أحضر الحلقة الأساسية وأُسجّل»، ومن لم تحضر الأساسية تُوسم في القائمة. تعرض واجهة الجلسة زرَّي المشرف «⏸️ إيقاف القائمة» (تجميد التسجيل) و«🛑 إنهاء الجلسة». وبعد التجميد يُغلق التسجيل الأساسي لكن يبقى **التسجيل الاحتياطي** متاحاً، وتظهر المسجّلات احتياطياً في قسم مستقل أسفل القائمة. يمكن للمشرفة تعديل حقلَي «حضور الأساسية» و«الاحتياط» لكل طالبة لاحقاً من `/classhistory`.

> **ضيوف الحلقة (walk-in) مع اعتماد لاحق:** في قوائم الحضور المرتبطة بقائمة الطالبات (الرئيسية والتدريب وتصحيح التلاوة)، إذا ضغطت طالبة غير مسجّلة زر الحضور فإنها تُحتسب حاضرة مباشرةً في الجلسة، وتُضاف كـ«طلب تسجيل معلّق» تعتمده المشرفة لاحقاً من `/pendingstudents` (الاعتماد يضيفها إلى قائمة المجموعة، وفي جلسات التدريب يُضاف اسمها كذلك إلى المجموعة الرئيسية المرتبطة). أمّا القوائم المفتوحة وجلسات الختمة فتُضيف الضيفة إلى القائمة مباشرةً.

## لوحة الإدارة الموحّدة (`/manage`)

تبقى قوائم الحضور الحيّة داخل المجموعة (تضغط عليها الطالبات)، بينما تُدار بقية الأدوات الإدارية من **لوحة واحدة تُفتح في المحادثة الخاصة مع المشرفة**. أرسلي `/manage` داخل المجموعة، فتصلك اللوحة في الخاص، ومنها تتنقّلين بين الأقسام بالأزرار (تُحرَّر الرسالة نفسها في مكانها):

- **👥 الطالبات** — لوحة `/students` (إضافة/حذف/تعديل).
- **⏳ طلبات الانضمام** — لوحة `/pendingstudents` (قبول/تجاهل).
- **🗂️ سجل الجلسات** — لوحة `/classhistory` (عرض التقرير أو تعديل السجلات).
- **👩‍🏫 المعلمات** — محرّر تفاعلي كامل: إضافة معلمات دفعةً واحدة (اختاري الدور أولاً ثم الصقي قائمة الأسماء)، تعديل الاسم، تعديل الأدوار (يمكن أن تحمل المعلمة أكثر من دور)، أو الحذف.
- **🏷️ مجموعات التدريب** — محرّر تفاعلي: إضافة مجموعة تدريب مرتبطة، إعادة تسميتها، حذفها، أو عرض طالباتها.
- **📚 المواد التعليمية** — كل مادة *درس* يحمل عنواناً وملفاً واحداً أو أكثر (مستند/صورة/فيديو/صوت): أرسلي أول ملف مع كتابة العنوان في التعليق، ثم أضيفي ملفات إضافية للدرس نفسه واضغطي **إنهاء الإضافة**. ويمكنكِ لاحقاً **إضافة ملف** إلى درس موجود، ثم **إرساله كاملاً إلى المجموعة** عند الحاجة أو حذفه (للمالكة والمشغِّلة).
- **📓 التكاليف** — متابعة تكليفات الطالبات في **مجموعة تكليف مخصّصة** مرتبطة بالحلقة: تعرض لكل تكليف عدد المسلِّمات والمُراجَعات وتفصيلاً لكل طالبة، مع زر **تنبيه غير المسلِّمات** بذكرهنّ في مجموعة التكليف.
- **🧑‍🏫 صفوف بدون مجموعة** — الانتقال إلى صفوفك الخاصة (`/offline`).

> تُسلَّم لوحات `/students` و`/pendingstudents` و`/classhistory` وكذلك لوحة `/manage` إلى المحادثة الخاصة مع المشرفة؛ فإن لم تكن قد بدأت محادثة مع البوت من قبل، ترسل لها المجموعة تنبيهاً بزر لفتح المحادثة عبر رابط `?start=manage`.

## متابعة التكاليف

يمكن ربط **مجموعة تكليف مخصّصة** بالحلقة الرئيسية، فيتتبّع البوت التسليمات والمراجعة تلقائياً داخلها:

1. تربط المشرفة المجموعة بالأمر `/addhomeworkgroup [رقم المجموعة]` (يُرسَل داخل الحلقة الرئيسية، ويجب أن يكون البوت عضواً في مجموعة التكليف). ولإلغاء الربط: `/removehomeworkgroup`.
2. تنشر المشرفة أو **معلمة التكليف** (`homeworkteacher`) رسالة التكليف داخل مجموعة التكليف مع الوسم **`#التكليف`**، فيُسجّلها البوت كتكليف جديد (يتفاعل بـ📓).
3. حين ترد طالبة **مسجّلة** في الحلقة على رسالة التكليف بحلّها، يُسجَّل تسليمها (يتفاعل بـ👍).
4. حين ترد المعلمة على تسليم الطالبة، يُعلَّم بأنه **روجِع** (يتفاعل بـ✅).

من زر **📓 التكاليف** في لوحة `/manage` تُتابع المشرفة لكل تكليف: عدد المسلِّمات والمُراجَعات وتفصيلاً لكل طالبة، مع زر **تنبيه غير المسلِّمات** بذكرهنّ في مجموعة التكليف. وتتوفّر متابعة يدوية مماثلة للصفوف الخاصة (بدون مجموعة).

## الصفوف الخاصة (بالمحادثة الخاصة مع البوت)

بجانب الحلقات داخل مجموعات Telegram، يمكن لأي أخت إدارة صفوفها الخاصة وتسجيل الحضور **بالمحادثة الخاصة مع البوت دون الحاجة لإضافته إلى مجموعة**. تُدار هذه الميزة بالكامل عبر أزرار تفاعلية، وتبدأ بالأمر `/offline`.

### ما الذي يمكن فعله؟

- **إنشاء صف** وإدارة قائمة طالباته (إضافة، تعديل اسم، حذف).
- **مجموعات التدريب داخل الصف:** إنشاء مجموعات (تسميات) داخل الصف، تعيين طالبة لمجموعة من قائمتها، وعرض طالبات كل مجموعة.
- **إدارة المعلمات:** إضافة معلمات دفعةً واحدة (اختاري الدور أولاً ثم الصقي الأسماء)، تعديل الاسم، تعديل الأدوار (يمكن أن تحمل المعلمة أكثر من دور: معلمة الحلقة / التدريب / التلاوة / التكاليف)، أو الحذف.
- **الجدول الأسبوعي (الروستر):** تعريف مواعيد أسبوعية متكررة للحلقات (الرئيسية / تصحيح التلاوة / التدريب / **مراجعة التكاليف**) بيومٍ ووقت، مع إمكانية إسناد معلمة لكل موعد. و**مراجعة التكاليف تمتد طوال اليوم** فتختارين لها اليوم فقط دون وقت (تظهر «طوال اليوم»). وللإسراع، زر **إضافة عدة مواعيد دفعةً واحدة**: اختاري النوع ثم الصقي أسطر «اليوم الوقت» (أو «اليوم» فقط للأنشطة اليومية). تعرضين الأسبوع كاملاً، ويجمع الأمر `/myweek` جدولكِ عبر كل حلقاتكِ. ويمكنكِ الضغط على أي موعد **لتعديل يومه أو وقته** أو إسناد معلمة له أو حذفه. ويمكن لكل مستخدمة ضبط **توقيتها المفضّل للعرض** و**بداية الأسبوع** دون الحاجة لتغيير توقيت الصف نفسه.
- **بدء الجلسات وتسجيل الحضور** بنفس أنواع القوائم (الحلقة الرئيسية، تصحيح التلاوة، التدريب) وتوليد تقرير لكل جلسة.
- **المواد التعليمية:** إضافة ملفات تعليمية (مستند/صورة/فيديو/صوت) بعنوان، ثم **إرسالها إليكِ** في المحادثة الخاصة عند الحاجة، أو حذفها (للمالكة والمشغِّلة).
- **التكاليف:** إنشاء تكليف بعنوان، ثم تسجيل حالة كل طالبة يدوياً بضغطة واحدة تتنقّل بين (لم تُسلِّم ⬜️ ← سلّمت 📝 ← صححت ✅) — بديل المتابعة التلقائية في مجموعة التكليف (للمالكة والمشغِّلة).
- **تعيين معلمة للجلسة** من زر «إدارة الجلسة»؛ ويظهر اسم المعلمة المعيّنة أعلى تقرير الجلسة.
- **مشاركة إدارة الصف** مع أخوات أخريات عبر أدوار محدّدة الصلاحيات.

### الأدوار والصلاحيات

| الدور | الصلاحيات |
|------|-----------|
| **المالكة** (منشئة الصف) | كل الصلاحيات: إعادة تسمية الصف، إدارة المشرفات وأدوارهنّ، القائمة والمعلمات والجلسات والتقارير. |
| **المشغِّلة (operator)** | كل العمليات **عدا** إعادة تسمية الصف وإدارة المالكة/المشغّلات؛ تشمل: إدارة القائمة والمعلمات والمواد التعليمية، بدء الجلسات وتعيين معلمة لها وحذفها، إضافة/إدارة المساعِدات، **ونسخ الصف المشترك إلى صفوفها الخاصة**. |
| **المساعِدة (assistant)** | تعديل الحضور في الجلسات القائمة وعرض التقارير فقط. |

### الدعوة والمشاركة

- تُنشئ المالكة أو المشغِّلة **دعوة للانضمام** تحتوي على رابط مباشر (deep link) بصيغة `https://t.me/<bot>?start=offline`.
- تفتح الأخت المدعوّة الرابط لبدء محادثة مع البوت، ثم تختار «🤝 صفوف شاركنني بها» للوصول إلى الصف.
- يمكن للمشغِّلة **نسخ الصف المشترك** إلى صفوفها الخاصة (يشمل النسخُ الطالبات والمعلمات دون سجلات الحضور).

> **البنية التخزينية:** يُخزَّن الصف الخاص كصف عادي في جدول `groups` مع معرّف اصطناعي (`telegram_chat_id = 'offline:<owner>:<uuid>'`) ومالكه في `owner_user_id`. تُحفَظ الطالبات في `members` والمعلمات في `teachers` (تحمل كل معلمة مجموعة أدوار في عمود `teacher_types text[]`)، وتُسجَّل المشرفات المفوَّضات في جدول `class_managers` (بدور `operator` أو `assistant`)، وتُحفَظ المواد التعليمية في جدول `class_materials` (يُخزَّن `file_id` من Telegram فقط دون رفع الملفات)، وتُحفَظ التكاليف وتسليماتها في جدولَي `homework` و `homework_submissions`، وتُحفَظ مواعيد الجدول الأسبوعي في جدول `class_schedule` وتفضيلات العرض لكل مستخدمة في `user_prefs`. ويحمل كل سجل جلسة حقل `teacher_id` للمعلمة المعيّنة.
